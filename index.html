Audio Player

<my-audio-player> — components/audioplayer.js
PLAYER

Composant autonome. Gère le graphe audio complet (MediaElementSource → Gain → Panner → Analyser), les contrôles de lecture et la couverture. Peut être chaîné via connectComponent().

Attributs

AttributTypeDéfautDescription
srcURL—URL du fichier JSON de playlist. Obligatoire. Réactif : changer l'attribut recharge la playlist.
autoplaybooleanabsentLance la lecture automatiquement au chargement (présence = true).

Format JSON de playlist

[ { "id": 1, "title": "…", "artist": "…", "audio": "…/track.mp3", "cover": "…/cover.png" } ]

Propriétés JS

PropriétéTypeAccèsDescription
currentTrackobject|nulllecturePiste chargée { id, title, artist, audio, cover } ou null.
audioCtxAudioContextlectureContexte audio partagé (singleton).

Méthodes

MéthodeSignatureDescription
connectComponent(target) → voidChaîne la sortie audio vers un autre composant ConnectableComponent.
getInputNode() → MediaElementAudioSourceNodeNœud d'entrée de la chaîne interne.
getOutputNode() → GainNodeNœud de sortie — point de connexion vers le composant suivant.

Événements émis

ÉvénementSurdetailDescription
track-changeddocument{ track, index, tracks[] }Émis à chaque changement de piste.
audio-playdocument—Émis quand la lecture démarre.
audio-pausedocument—Émis quand la lecture se met en pause.

Événements écoutés

ÉvénementSurdetailDescription
track-selectdocument{ index }Sélectionne et lance la piste à l'index donné.
track-importdocument{ track }Ajoute une piste importée à la liste interne.
playlist-cleardocument—Vide la liste et arrête la lecture.

Graphe audio interne

<audio> → MediaElementSourceNode → GainNode (vol) → StereoPannerNode → AnalyserNode (VU) → outputNode → ctx.destination

Playlist

<my-playlist> — components/Playlist.js
PLAYLIST

Composant autonome, UI pur. Aucun graphe audio. Se synchronise avec <my-audio-player> exclusivement via événements sur document — les deux peuvent fonctionner indépendamment.

Attributs

AttributTypeDéfautDescription
Aucun attribut observé. Composant entièrement piloté par événements.

Fonctionnalités

FonctionnalitéDescription
AffichageCover miniature, titre, artiste, durée par piste, durée totale.
Drag & dropRéordonnancement des pistes par glisser-déposer.
Import fichierBouton "+ Fichier" ou drag & drop sur le composant.
Import URLBouton "+ URL" — saisie d'une URL distante.
ClearVide la playlist et émet playlist-clear.

Événements émis

ÉvénementSurdetailDescription
track-selectdocument{ index }Clic utilisateur sur une piste.
track-importdocument{ track }Piste importée (fichier ou URL).
playlist-cleardocument—Playlist vidée par l'utilisateur.

Événements écoutés

ÉvénementSurdetailDescription
track-changeddocument{ track, index, tracks? }Met à jour la liste complète si tracks est présent ; met en surbrillance la piste active dans tous les cas.

Equalizer

<my-eq> — components/equalizer.js
EQUALIZER

Composant autonome et chaînable. 6 filtres BiquadFilter de type peaking en série. Fonctionne seul (sortie → ctx.destination) ou inséré dans une chaîne via connectComponent().

Attributs

AttributTypeDéfautDescription
presetstring"0,0,0,0,0,0"Gains dB des 6 bandes séparés par virgule. Ordre : 60Hz, 170Hz, 350Hz, 1kHz, 3.5kHz, 10kHz. Plage : −30 à +30 dB. Réactif.

Préréglages intégrés

NomValeur
Flat0,0,0,0,0,0
Bass5,4,2,0,-1,-2
Treble-2,0,2,4,3,1
Rock4,3,0,-2,2,3
Pop3,2,0,1,3,4
Jazz2,0,-1,0,2,4
Vocal0,0,0,0,-3,-5

Méthodes

MéthodeSignatureDescription
connectComponent(target) → voidChaîne la sortie du dernier filtre vers un autre composant.
getInputNode() → BiquadFilterNodePremier filtre (60Hz) — point d'entrée.
getOutputNode() → BiquadFilterNodeDernier filtre (10kHz) — point de sortie.

Graphe audio interne

→ BiquadFilter(60Hz) → BiquadFilter(170Hz) → BiquadFilter(350Hz) → BiquadFilter(1kHz) → BiquadFilter(3.5kHz) → BiquadFilter(10kHz) → ctx.destination

Waveform

<my-waveform> — components/waveform.js
WAVEFORM

Composant autonome et chaînable. Passthrough audio transparent — lit le signal sur un AnalyserNode pour visualiser la forme d'onde ou le spectre, sans modifier le signal.

Attributs

AttributTypeDéfautDescription
colorstring#39e082Couleur du tracé (valeur CSS). Réactif.
fft-sizenumber1024Taille FFT de l'AnalyserNode (puissance de 2, 32–32768). Réactif.
modewaveform|spectrumwaveformMode de visualisation : forme d'onde temporelle ou spectre fréquentiel. Réactif.

Méthodes

MéthodeSignatureDescription
connectComponent(target) → voidRarement utilisé — ce composant est généralement le dernier avant le visualiseur.
getInputNode() → AnalyserNodeNœud d'entrée et de sortie (passthrough).
getOutputNode() → AnalyserNodeMême AnalyserNode.

Graphe audio interne

→ AnalyserNode → ctx.destination └─ getByteTimeDomainData() / getByteFrequencyData() → canvas (requestAnimationFrame)

Butterchurn (Milkdrop)

<my-butterchurn> — components/butterchurn.js
VISUALIZER

Composant autonome et chaînable. Passthrough audio. Alimente le visualiseur Milkdrop (Butterchurn) via un AnalyserNode. Le rendu canvas est actif uniquement pendant la lecture (audio-play / audio-pause).

Attributs

AttributTypeDéfautDescription
blendnumber2.0Durée de transition en secondes entre les presets Milkdrop. Réactif.

Méthodes

MéthodeSignatureDescription
connectComponent(target) → voidRarement utilisé — généralement dernier de la chaîne.
getInputNode() → GainNodeNœud d'entrée.
getOutputNode() → AnalyserNodeNœud de sortie (après le gain).

Événements écoutés

ÉvénementSurDescription
audio-playdocumentActive le rendu canvas Milkdrop.
audio-pausedocumentSuspend le rendu canvas.

Graphe audio interne

→ GainNode → AnalyserNode → ctx.destination └─ butterchurn.connectAudio(gainNode) → canvas (requestAnimationFrame)
